# MarketMamba 雙模型架構重整計畫（v2）

> 目的：將現有單一模型（V6 系列）拆分為「廣度模型」與「高信念模型」兩套**方法論完全不同**的獨立系統，各自完整（預測／回測／版本），並補齊敘事性文字內容，作為 UC Capital 實習申請作品 + 個人實盤操作參考。
>
> v2 相較 v1 的核心修正：conviction 線不再把 DL Alpha 當共識評分的其中一票，而是重新定位 DL／LLM 在兩條線裡的角色——**DL 是 breadth 的核心引擎，是 conviction 的工具層**。本文件為頁面架構 + 內容規格，實際 codebase 對應留待你對照現況決定。

---

## 0. 核心定位敘事（首頁需要的文字）

兩條線的分野依據 Grinold 主動管理基本法則：

> **IR ≈ IC × √Breadth**（資訊比率 ≈ 資訊係數 × 廣度平方根）

同樣的總主動報酬，可以用「更多標的、較低單檔信心」取得，也可以用「更少標的、更高單檔信心」取得。但這只解釋了「押注方式」的差異，還沒解釋「方法論」為什麼要跟著換——這是 v2 補上的第二層邏輯：

**breadth 的 edge 是統計性、可重複的模式，散布在幾千檔上——這正是深度學習（DL）擅長的 pattern matching。conviction 的 edge 來自對少數特定情境的深刻理解——樣本數太少，訓練/驗證一個深度網路只會過擬合到毫無意義。** 所以兩條線不只是資金規模不同，核心引擎的本質就不同：一個是統計學習驅動，一個是判斷驅動、LLM 輔助研究。

| | 廣度模型 | 高信念模型 |
|---|---|---|
| 標的範圍 | 全市場 ~2,888 檔 | 精選 10–20 檔 |
| 核心假設 | 統計顯著性來自覆蓋面（√Breadth） | 準確性來自對少數標的的深度研究 |
| 核心引擎 | Mamba SSM + GATv2（DL 是決策主軸） | 量化篩選 + LLM 研究增幅 + 人工判斷（DL/LLM 是工具層） |
| 資金定位 | Paper／驗證用，不代表個人實際部位 | 個人實盤操作主要依據 |
| 可回測性 | 可嚴謹 Walk-Forward（12 年結構化數據） | 部分可回測（事件驅動子引擎），主體只能前瞻追蹤 |
| 主要用途 | 作品集：證明能做機構規模化選股系統 | 個人投資參考 + 展示「知道什麼時候不該用 DL」的方法論成熟度 |

首頁需要一段白話文字：「為什麼要開兩個模型」——不是因為懶得整合，是因為兩者的 edge 來源本質不同，用同一套工具硬做兩件事，兩邊都會做不好。

## 2. 高信念量化模型 `/conviction`（個人操作主軸）

### 2.0 模型簡介區塊
文字說明重點：
- 這條線解決的問題：資金有限，無法分散到全市場，要在少數標的上做對決定
- **明確聲明方法論分工**：DL Alpha 不是排序主力，只在篩選層當輔助訊號／風險檢查；核心判斷來自量化篩選 + LLM 研究整理 + 人工最終定奪
- 誠實聲明：這條線不假裝有金融背景撐起的深度基本面分析，能做到的是把研究流程結構化、可審核、可累積經驗——這本身就是 UC Capital 要的 AI 工程能力展示，不是財經專業展示
- 跟廣度模型的關係：資料源部分重疊，方法論刻意不同

### 2.1 三層引擎結構（技術核心，需要在頁面上圖解）

**篩選層（量化，低成本）**
從全市場用規則式篩選縮小到 30–50 檔候選：估值分位數、法人買賣超持續性、事件行事曆（財報/法說會/除權息）。DL Alpha 在此層只做輔助訊號或尾部風險標記，不參與排序。

**研究層（LLM，成本控制設計）**
- **觸發機制**：不排程每日全跑，只在候選有變化時觸發——新進候選池、法人買賣超轉向、財報公布、股價異常波動。大幅降低呼叫量。
- **先做單一結構化 prompt，不要一開始上多智能體**：一次呼叫依序輸出估值判讀、催化劑、籌碼與情緒交叉檢查、風險檢查四個段落，成本約為四支獨立 agent 的四分之一，可拿到多智能體大部分的效果。等真的需要 agent 互相辯論或需要不同工具（例如查歷史事件庫）再拆分。
- **分層用模型**：篩選/粗判用便宜模型，只有進入候選池的標的才用較貴模型生成完整 thesis。
- 版本紀錄裡可以記這條路線：v1 單一 prompt → v2 拆分 agent（等預算允許），本身是一個可講的工程決策敘事。

**整合層（人工 + PM 角色）**
把研究層輸出收斂成一張 thesis 卡片：核心押注邏輯、信念分級（S/A/B）、部位大小建議、**證偽條件**（先寫下什麼情況代表這個 thesis 錯了，逆風時才有依據，而不是硬撐）。

### 2.2 子頁：每日訊號
- 信念分級表（S/A/B）+ 觸發事件標記（今天為什麼重新評估）
- 原 Scanner 的進場/觀察清單/退場、Trailing Stop 機制保留

### 2.3 子頁：模型預測結果 → 改為「Thesis 卡片牆」
- 每檔一張卡片：thesis 摘要、催化劑、估值判讀、風險檢查、DL 輔助訊號（小字附註，非主要呈現）、證偽條件
- 型態學雙重確認結果併入卡片作為技術面佐證

### 2.4 子頁：回測結果 → 誠實拆兩塊
- **可嚴謹回測的部分**：如果研究層發現的候選中有事件驅動性質的子策略（財報意外後動能延續、法人籌碼轉向反轉、除權息規律），這些有足夠歷史事件數、結構化數據，可以真的做 Walk-Forward，明確標示為「conviction 線底下的量化子引擎」
- **無法回測、只能前瞻追蹤的部分**：明確標示「即時追蹤中（自 XXXX-XX-XX 起）」，呈現方式見下方研究日誌

### 2.5 子頁：版本紀錄
記錄兩條軌跡：
- 篩選規則的變更（門檻調整、新增訊號）
- Agent pipeline 的演進（prompt 版本、單一 prompt → 多智能體的過渡、模型分層策略調整）

### 2.6 子頁：持倉追蹤
從現有 `/portfolio` 移入，對應真實永豐證券帳戶，保留現況功能。

### 2.7 研究日誌（新增，貫穿 2.3–2.4 的資料骨幹）

這不只是給人看的展示頁，是**校準工具**——半年、一年後可以回頭檢驗「S 級 thesis 的實際報酬有沒有顯著贏過 B 級」，這比任何回測都更誠實地告訴你這套流程有沒有訊號。

建議欄位（不管最後存在 conviction 頁面的資料庫還是 Obsidian，先定規格）：

| 欄位 | 說明 |
|---|---|
| thesis_id | 唯一識別碼 |
| 產生時間 | thesis 生成的時間戳 |
| 標的 | 股票代號/名稱 |
| 觸發原因 | 為什麼這次重新評估（新進候選/法人轉向/財報/異常波動） |
| 量化篩選分數 | 進入候選池當下的篩選層分數 |
| 研究層輸出摘要 | LLM 生成的四段論述精簡版 |
| 信念分級 | S/A/B（人工最終定奪） |
| 進場價位/日期/部位大小 | 若實際進場 |
| 證偽條件 | 先寫下什麼情況代表判斷錯誤 |
| 目前狀態 | 追蹤中/已出場/已證偽 |
| 結果與事後檢討 | 出場後回填，包含「證偽條件有沒有正確預示問題」 |

這份日誌從第一筆 thesis 開始就要用同一個格式記錄，資料才會乾淨地累積，不要想到才回頭補。

## 3. 廣度量化模型 `/breadth`（作品展示主軸，DL 是核心引擎）

### 3.0 模型簡介區塊
- 明確定位：這是機構等級的量化做法，DL 在這裡才是決策核心，因為訓練訊號來自 ~2,888 檔 × 12 年的海量橫斷面樣本，撐得起深度網路
- 架構亮點：Mamba SSM + GATv2 知識圖譜
- 明確標註：此模型用於驗證方法論與市場實測，非個人資金實際操作對象（資金規模不足以分散到全市場）

### 3.1 子頁：模型預測結果
- 全市場 Alpha 排名（Top 50，可展開）
- 產業強弱熱力圖
- **新增：知識圖譜視覺化**——互動式節點圖（產業/集團/供應鏈/相關性邊），目前完全沒展示但技術辨識度最高，優先做

### 3.2 子頁：回測結果
- 分年 IC/ICIR 分佈（Walk-Forward 各 Fold）
- 因子歸因：Alpha 來自 Group A 動能 / B 籌碼 / C 基本面 / D 總經各自的邊際貢獻
- 換手率與容量分析（流動性門檻內可交易檔數，模擬機構規模資金的可執行性）
- 累積超額報酬 vs TAIEX

### 3.3 子頁：版本紀錄
Model Card 格式，記錄 V6.0 → V6.3 的演進（沿用既有 PR 紀錄）。

### 3.4 子頁：雙模型比較
原「雙模型」頁面移入（待確認實際內容範圍：V6.2 vs V6.3，或 pure-quant vs quant+LLM），作為廣度模型內部的 A/B 測試工具。
