## 二、模型架構財金視角評估

### 2-1. Mamba SSM 的財金意義

Mamba 的**選擇性狀態空間機制**在財金應用中具有獨特優勢，值得詳細討論：

```
傳統 Transformer：
  每個時間步都「看」所有歷史 → 會被不相干的歷史干擾
  → 在股市中相當於：2012 年的量價特徵影響 2024 年的判斷

Mamba SSM：
  動態「遺忘門」→ 自動選擇哪些歷史值得記住
  → 在股市中相當於：動態辨識哪些歷史形態與當前相似
```

**財金評估**：這個設計方向 **正確**。台股在不同市況下（牛市/熊市/盤整）的有效因子差異極大，一個能「動態選擇記憶」的架構，理論上能比固定注意力的 Transformer 更好地適應市場狀態切換。

### 2-2. 多尺度三路 Mamba 設計

```
Short Branch (20d) → 捕捉短線動能 + 籌碼面
Mid Branch   (60d) → 捕捉季線趨勢 + 外資中期佈局
Long Branch  (252d) → 捕捉基本面週期 + 年線模式
```

**財金評估**：這三個時間尺度的選擇 **極具財金意義**：

- **20 天**：正好是台股一個交易月，對應月線 KD 週期、月結後的資金流動
- **60 天**：季線，對應法說會週期、EPS 預期修正週期
- **252 天**：年線，對應完整的財報週期、年度動能因子（12-1 動能）

> [!TIP]
> 這個多尺度設計可類比為同時在看「5 分鐘、60 分鐘、日線」的多週期分析，但用機器學習自動融合，比人工判斷更系統化。

### 4-2. Trailing Stop 機制評估

```
< +5%  → 固定 -5%（成本價）
≥ +5%  → 成本 +2%（鎖利）
≥ +10% → 成本 +6%
≥ +15% → 成本 +10%
```

**財金評估**：

✅ **設計思路正確**：Trailing Stop 的核心邏輯是「讓獲利奔跑，及時止損」，這與台股個股波動率（平均日波動 1.5~3%）配合得當。

⚠️ **潛在問題**：

1. **初始 -5% 止損過寬**：對於 Alpha_20d 預期 +15~25% 的標的，-5% 的初始止損相當於損益比 1:3，是合理的。但對於 Alpha_20d 只有 +8~10% 的標的，-5% 止損讓期望值大幅壓縮。
2. **缺乏時間止損**：台股有些標的「不漲也不跌」，若持倉超過 10 天未達 +5%，是否應強制出場？目前設計未涵蓋。
3. **缺乏大盤崩跌的緊急止損**：當 TWII 單日 -3% 以上時，個股止損線應緊縮，但目前機制未處理此情境。

### 5-2. 最嚴重的缺口：組合層面風控

目前所有風控都是**個股層面**的，缺乏**組合層面**的保護機制：

```
場景：
  持有 8 支股票，個別 Sharpe 都 > 3
  但如果這 8 支都是 AI 伺服器供應鏈（高相關性）
  
  發生：
  Nvidia 財報爆雷 → 台股 AI 鏈全線崩跌
  → 8 支同步觸發 -5% 止損
  → 組合單日 -5%（實際上接近 -5%，因所有持倉同步止損）
```

> [!CAUTION]
> **缺乏組合 VaR / CVaR 監控**。在極端市況下，個股 Trailing Stop 的同步觸發不能防止組合層面的大幅回撤。建議加入：
> - 組合 5 日最大回撤 ≥ -8% → 全部倉位降至 50%
> - 組合 5 日最大回撤 ≥ -12% → 清倉觀望

### 6-2. ~~Sharpe Score~~ → Signal_Quality 計算方式的問題 ✅ 命名已修正 (I2)

> ✅ **命名已修正（2026-05-25）(I2)**：全域將 `Sharpe_Score` 欄位重命名為 `Signal_Quality`（影響 9 個檔案），更準確反映此指標為「Alpha 信噪比」而非傳統 Sharpe Ratio。計算公式不變，仍為 `Net_Alpha_20d / (Uncertainty + 1e-6)`。

從代碼觀察到的 Signal_Quality 計算：

```python
Net_Alpha_20d = Exp_Alpha_20d - Slippage_Est
Signal_Quality = Net_Alpha_20d / Uncertainty  # 原名 Sharpe_Score，已重命名
```

**問題**：這個公式是 Alpha / Uncertainty 的比值，實際上更接近「信噪比」而非傳統 Sharpe Ratio。

**財金問題**：
1. 分母是預測不確定性，而非實際報酬標準差 → 無法與傳統 Sharpe 比較
2. 6818 的 Sharpe = 6.105 / 1.118 = 5.46 → 看起來很高，但 Uncertainty = 1.118 代表預測區間包含正負 100%，幾乎毫無意義

**建議**：改用 `Sharpe = Net_Alpha_20d / (Uncertainty × sqrt(20))` 作為年化調整，或直接用歷史實際報酬的 Sharpe。

  因子研究 → IC 驗證 → Walk-Forward → 風險調整 → 小額實盤 → 放大

MarketMamba 現狀：
  因子研究 ✅ → IC 驗證（部分）⚠️ → Walk-Forward ❌ → 風險調整 ⚠️ → 小額實盤 ✅（融程電）

**結論**：直接從「因子研究」跳到「小額實盤」，跳過了 Walk-Forward 這個最關鍵的誠實考試，有過擬合風險。

## 八、關鍵缺口與改進建議

### 🔴 P0（實盤前必須修復）

#### ~~問題 1：異常 Alpha 值未過濾~~ ✅ 已完成 (S1)

> ✅ **已完成（2026-05-25）**：`run_inference()` 已加入 `Exp_Alpha_5d/20d/60d.clip(-2.0, +2.0)` 截斷，`Uncertainty` 截斷至 ≥0。

**現象**：6818（Alpha_20d = +610.9%）、7818（+276.7%）等異常值通過所有過濾器進入推薦名單。

**根因**：模型對 OOD（Out-of-Distribution）樣本沒有防護機制，Sharpe Score 用異常的 Net_Alpha 作分子，放大了這些股票的排名。

**修復方案**：
```python
# 在 compute_kelly_sharpe() 中加入 Alpha 合理性截斷
ALPHA_20D_CAP = 0.5  # 月報酬不可能超過 50%（除非短暫特殊行情）
df['Net_Alpha_20d'] = df['Net_Alpha_20d'].clip(upper=ALPHA_20D_CAP)

# 或：用百分位截斷（更穩健）
p99 = df['Exp_Alpha_20d'].quantile(0.99)
df.loc[df['Exp_Alpha_20d'] > p99, 'Sharpe_Score'] = -999.0
```

#### 問題 2：Walk-Forward 驗證缺席

**現象**：所有「策略績效」均基於 Quick Fold（N=500，15 epoch）結果，沒有跨時期驗證。

**修復方案**：依照 V6_Master_Plan.md 中的規劃，執行 36 個 Fold 的 Expanding Walk-Forward，這是實盤前的最高優先級任務。

**最低要求**：在開始正式操作前，至少完成 **4 個 Fold**（2019-2020 訓練 → 2021 測試，etc.），確認 IC 在不同市況下均為正值。

### 🟡 P1（重要改進，建議在擴大部位前完成）

#### ~~問題 3：Uncertainty 閾值校準問題~~ ✅ 已完成 (I1)

> ✅ **已完成（2026-05-25）**：`scanner.py` 的 `_check_confidence()` 已改為動態閾值，以當日全市場 `Uncertainty` 的 **30th percentile** 作為「高信心」門檻（每日 `run_scan()` 開始時計算），不再使用固定的 0.02。

**現象**：`Uncertainty < 0.02` 在當前模型下幾乎不可能滿足，條件 2 等同虛設。

**改進方案**：
```python
# 動態閾值：以當日全市場 Uncertainty 分布的 25th percentile 作為「高信心」門檻
uncertainty_25th = df['Uncertainty'].quantile(0.25)
condition_2_met = df['Uncertainty'] < uncertainty_25th  # 動態閾值
```

#### 問題 4：缺乏組合層面的 Beta 中性控制

**現象**：推薦標的可能高度相關，在大盤暴跌時同步觸發止損。

**改進方案**：計算每支推薦股票相對 TWII 的 Beta，限制組合整體 Beta 在 0.8~1.2 之間。若 Beta 過高，主動選入 Beta 較低的標的平衡。

#### 問題 5：基本面動量因子缺乏時間對齊防護

**現象**：月營收（每月 10 日公布）、EPS（季報）等基本面數據有「資訊時間差」，若沒有正確的 point-in-time 對齊，可能存在前瞻偏差（Look-Ahead Bias）。

**改進方案**：確認特徵工程中所有基本面數據均使用「公告日+1 天」作為起效日，而非「財報截止日」。

---

### 🟢 P2（中長期優化建議）

#### 建議 1：加入 Fama-French 因子剝離

目前 Alpha 目標是「相對 TWII 超額報酬」，未剔除 Size（小公司效應）和 Value（低 PBR 效應）因子。這導致模型可能只是在捕捉「小型股超額報酬」而非真正的 Alpha。

建議按 V6_Master_Plan 中的規劃，建立台股 FF4 因子，對 Alpha 目標進行因子剝離後再訓練。

#### 建議 2：加入截面 z-score 標準化後的 IC 計算

目前 IC 計算方式需確認是 Pearson IC 還是 Spearman IC。業界更偏好 **Rank IC（Spearman）**，因為它對異常值更穩健：

```python
from scipy.stats import spearmanr

def compute_rank_ic(pred: pd.Series, actual: pd.Series) -> float:
    """截面 Spearman 相關係數"""
    return spearmanr(pred, actual)[0]
```

#### 建議 3：引入因子衰減分析

每個因子的預測力都有「半衰期」，建議分析：
- 在預測日 +1, +5, +10, +20, +60 天後，IC 是否持續為正？
- 如果 IC 在 +5 天後已接近 0，說明 5d 預測可靠但 20d 預測可能是噪音。
