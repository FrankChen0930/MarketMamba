### 步驟概覽

```
本機跑爬蟲 (選配)     Colab 推論管線
    │                    │
    │                    ├── Cell 1: 環境建置
    │                    ├── Cell 2: 資料同步
    │                    ├── Cell 3: 特徵工程
    ├──上傳Drive──→     ├── Cell 4: 情緒引擎 (選配)
                         ├── Cell 5: AI 推論
                         ├── Cell 6: 型態掃描
                         ├── Cell 7: 調倉
                         └── Cell 8: GitHub 推送 → Streamlit 自動更新
```

### 詳細步驟

#### 1️⃣ (選配) 本機跑新聞爬蟲

```powershell
cd D:\Desktop\work\MarketMamba\MarketMamba
python scripts/crawl_local.py --days 30 --workers 4

# 如果本機有 GPU 或想用 CPU 跑 FinBERT：
python scripts/crawl_local.py --days 30 --run-finbert
```

完成後，將產出的 `News_Cache/` 資料夾上傳到 Google Drive：
```
MyDrive/MarketMamba_V5/News_Cache/
```

> 💡 日常推論時，如果不想在本機跑爬蟲，也可以直接在 Colab 即時爬取
> （新聞量少，幾分鐘就跑完）。只有訓練時建議用本機多核心爬蟲。

#### 2️⃣ Colab 推論

1. 打開 Google Colab → 新增 Notebook
2. Runtime → Change runtime type → **GPU (A100)**
3. 打開 `notebooks/V5_5_Pipeline.py`
4. 把每個 `# %% Cell` 區塊**分別**複製貼入 Colab 的獨立 Cell
5. 從 Cell 1 開始依序執行

**如果不需要情緒引擎**：直接跳過 Cell 4，系統會自動偵測到沒有情緒特徵，
退回 V5.0 模式（46 維），使用現有的 `V5_DynamicGAT_Production.pth` 模型。

**如果要啟用情緒引擎**：取消 Cell 4 的註解。
⚠️ 第一次啟用時，必須先跑一次 `V5_5_Training.py` 訓練 84 維的新模型。

## 🧠 訓練新模型

### 什麼時候需要重訓？

| 時機 | 原因 |
|------|------|
| 從 V5.0 升級到 V5.5 (啟用情緒引擎) | input_dim 46 → 84，舊權重不相容 |
| 累積了新的訓練資料 (每季建議) | 讓模型學習最新市場規律 |
| 市場結構發生重大變化 | 如：新的交易制度、重大政策 |

### 訓練步驟

1. 打開 `notebooks/V5_5_Training.py`，同樣貼入 Colab
2. **Cell 1**：環境建置（自動偵測 A100 並設定 batch_size=256）
3. **Cell 2**：資料同步 + 特徵工程
4. **Cell 3**：（選配）載入本機預爬的新聞快取
5. **Cell 4**：調整訓練參數，開始訓練
6. **Cell 5**：看 Loss 曲線，判斷是否過擬合
7. **Cell 6**：用新模型跑推論驗證
8. **Cell 7**：推送結果到 GitHub

### 訓練參數建議

| 參數 | A100 (40GB) | T4 (15GB) | 說明 |
|------|------------|-----------|------|
| batch_size | 256 | 64 | 系統自動偵測 |
| epochs | 50 | 50 | Early Stopping 會自動提前結束 |
| lr | 1e-4 | 1e-4 | 搭配 Cosine Annealing |
| early_stop | 7 | 7 | 連續 7 epoch 無改善就停 |

## 📰 消息面情緒引擎 — 細節說明

### 新聞來源

| 語言 | 來源 | 覆蓋範圍 |
|------|------|---------|
| 英文 | Google News RSS | 全球財經、地緣政治 |
| 英文 | Yahoo Finance RSS | 美股即時 |
| 英文 | Finviz RSS | 量化圈熱門 |
| 中文 | Google News RSS (TW) | 台股個股、大盤 |

所有來源經過：
1. **白名單過濾**：只保留 Reuters、Bloomberg、鉅亨網等高品質來源
2. **標題去重**：相似度 > 70% 的重複報導自動過濾
3. **FinBERT 分析**：雙模型（英文 ProsusAI + 中文 chinese-finbert）

### 多來源不會造成混亂嗎？

**不會。** FinBERT 的 embedding 平均機制天然適合多來源：

| 情境 | 多來源 embedding 平均的效果 |
|------|---------------------------|
| 各家一致看多 | 分數強化 → 強共識訊號 |
| 觀點分歧 | 正負抵消 → 反映市場分歧（本身就是有價值的資訊）|
| 只有一家報導 | 單一來源權重自然較低 |

### 訓練時的新聞怎麼辦？（歷史資料問題）

Google News RSS 只能抓到最近 ~30 天的新聞。
2019~2025 的歷史訓練資料**不會有情緒特徵**。

**這是正常且可接受的：**
- 歷史資料的情緒欄位全部 = 0
- 模型會學到：「情緒 = 0 → 沒有消息面訊號 → 只看量價籌碼」
- 近期有新聞的資料：「情緒 ≠ 0 → 有消息面訊號 → 加權考慮」
- 隨著每天累積新聞，有效訓練資料會越來越多

### Auto-Labeling（自動標籤）

系統會自動用**股價反應**來標註新聞好壞：
- 新聞發布後 5 天的累積 Alpha > +1% → Positive
- Alpha < -1% → Negative
- 其他 → Neutral

累積 ≥ 1000 條標註後，可執行 `auto_labeler.py` 的 `finetune_finbert()` 微調模型。

## 🔧 V5.0 → V5.5 的差異

| 項目 | V5.0 (原版) | V5.5 (新版) |
|------|------------|------------|
| 程式碼 | 1077 行 Notebook | 30 個模組化 .py 檔案 |
| 輸入維度 | 46 維 (量價籌碼) | 84 維 (+38 情緒特徵) |
| 圖建構 | cosine similarity KNN | KG-Enhanced (cosine + 產業分類) |
| 消息面 | ❌ | ✅ 雙軌 FinBERT (EN+CN) |
| 季報對齊 | +90 天粗估 | IFRS 法定截止日 |
| 錯誤處理 | `except: pass` | `logger.warning()` + 分類策略 |
| 帳本日期 | 混亂格式 | 統一 YYYY-MM-DD |
| 訓練模組 | 無 | ✅ trainer.py (tqdm + Early Stop) |

### 向下相容

推論引擎會**自動偵測**特徵矩陣，無需手動切換：
- 有情緒特徵 → V5.5 模式 (84 維, KG-GAT)
- 沒有情緒特徵 → V5.0 模式 (46 維, cosine KNN)

**現有的 `V5_DynamicGAT_Production.pth` 可以直接使用。**

## ❓ 常見問題

### Q: 我需要在本機安裝 PyTorch 嗎？

不需要。本機只用來寫程式碼和跑新聞爬蟲。
爬蟲不需要 PyTorch（除非你想在本機跑 FinBERT，加 `--run-finbert`）。

### Q: Mamba 的 wheel 是 A100 專用嗎？

不是。`mamba_ssm-2.3.0-cp312-cp312-linux_x86_64.whl` 是 Linux + CUDA 通用版，
A100、V100、T4 都能用。

### Q: 情緒引擎每次推論都要重新爬蟲嗎？

是的，日常推論時 Cell 4 會即時爬取最新新聞。
但因為只爬最近 3 天的新聞，量很少，通常幾分鐘就完成。

### Q: 可以不要自動關機嗎？

可以。把 `notebooks/V5_5_Pipeline.py` 最後一行 `shutdown_colab()` 註解掉就好。

### Q: Git push 失敗怎麼辦？

確認你在 Colab 的 Secrets 裡設置了 `GITHUB_TOKEN`：
1. 左邊側邊欄 → 🔑 Secrets
2. 名稱: `GITHUB_TOKEN`，值: 你的 GitHub Personal Access Token
3. 權限需要勾選 `repo`

### Q: 「RuntimeError: CUDA out of memory」怎麼辦？

降低 `batch_size`。A100 一般不會遇到這問題，但如果同時跑了多個 Notebook 就可能。
