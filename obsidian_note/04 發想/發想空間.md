## 前端整合構想：嵌入即時看盤 / 下單（參考 shioaji-pro-app）

> 來源：[Sinotrade/shioaji-pro-app](https://github.com/Sinotrade/shioaji-pro-app)，2026-06-12

**痛點**：現在看到 Model 推薦的股票後，還要切去券商 App / 看盤軟體查即時報價、K 線、決定要不要下單，流程斷在兩個地方。

**shioaji-pro-app 是什麼**：永豐金官方做的 React+TS 前端，直接打本機跑的 `shioaji server`（Python，localhost:8080），功能包含即時 SSE 報價、K 線（lightweight-charts）、五檔、點圖下單、停損停利觸價單、可拖拉版面。重點是**零後端**——前端直接連本機 shioaji server。

### 可行性分層

1. **最小可行（讀取為主）**：在 PersonalOS 的選股/Scanner 頁面，點開一支推薦股票時，內嵌即時報價 + K 線（用 `lightweight-charts`，跟 shioaji-pro-app 同一套）。資料來源可以是 Shioaji API（user 已有帳戶+API Key，`PersonalOS` 也已串永豐持倉同步）或現有的 yfinance/FinMind 報價。
   - 好處：解決「看到推薦還要切 App」的主要痛點，且不涉及下單，風險低
   - 限制：即時 tick 需要本機跑 shioaji server（Electron app 可以當 sidecar process 啟動），純 PWA/Web 版做不到即時 tick

2. **中度整合（五檔 + 下單面板）**：仿 shioaji-pro-app 的下單面板（整股/零股、ROD/IOC/FOK），但維持「Claude/系統不能下單」的原則——這是給使用者自己手動操作用的 UI，不是自動交易
   - PersonalOS 是 Electron，可以整合 shioaji server 為 sidecar，把 shioaji-pro-app 的元件（react-grid-layout + lightweight-charts + vanilla-extract）拆過來改裝成「MarketMamba 風格」的下單面板
   - 工作量：中～大，需要先確認 shioaji server 在背景常駐執行的穩定性

3. **完整版（點圖下單、停損停利觸價單）**：直接把 shioaji-pro-app 整套接進來當「進階模式」分頁，PersonalOS 提供「選股建議」，這個分頁提供「下單執行」，兩者用同一份持倉資料同步
   - 工作量：大，且要處理金鑰管理、模擬/正式環境切換等安全性問題

**建議切入點**：先做第 1 層（K 線+即時報價內嵌），這個直接打中「看到推薦還要切 App」的核心痛點，技術風險也最低。下單相關（第 2、3 層）可以之後再評估，且無論做到哪一層，下單動作都應該是「使用者手動點擊確認」，不會做成自動化。

### 範圍確認（2026-06-12）

- **不做交易/下單**：使用者明確表示不打算把交易功能做進系統，太危險。這個構想**只做「看盤」**——把即時報價/K線拉進來，純展示。
- **只能在 PersonalOS（電腦版）做，網頁版/手機 PWA 做不到**：
  1. `shioaji server` 跑在使用者電腦本機（`localhost:8080`，帶 API Key），手機/Vercel 網頁物理上連不到——除非對外網開放，但那比下單風險更高，不考慮
  2. 即使是同一台電腦的瀏覽器，Chrome 的 Private Network Access 政策也會擋 https 網頁打 `http://localhost`；Electron（PersonalOS）不受此限制
- **定案**：即時報價+K線內嵌 = PersonalOS 桌面版專屬功能（純看盤、view-only）；手機 PWA 維持現有延遲報價即可，分工合理（選股決策本來就適合在電腦上做）

## AI 量化策略影片整理（爸傳的，僅供參考）

> 來源：YouTube「用AI產出冠軍量化交易策略」整理，2026-06-12，使用者尚未細看

核心內容是 XQ 全球贏家 + AI 轉碼的工作流，跟 MarketMamba 的技術路線（自己訓練 Mamba+GATv2 模型）不是同一條路，比較像零基礎玩家的捷徑，價值不大。

**唯一可能有共鳴的點**：影片提到「2023 量化積木冠軍策略」回測報酬 222%，但實戰致命傷是「下午 1 點漲停鎖死才買」根本買不到，是典型 overfitting / 看得到吃不到。正確做法是在「即將突破、尚未鎖死前」就佈局。

這個教訓跟你 [[投資策略框架]] 裡 2072 世紀風電案例的邏輯是一致的（題材+型態確認就進場，不是等漲停鎖死）。但 MarketMamba 是**收盤後**的日頻推論，不存在「盤中追漲停」的問題，所以這個教訓對模型本身沒有直接適用性，純粹是再次驗證你現有策略框架的方向是對的。

結論：這份筆記本身不需要特別融入系統，可以歸檔備查即可。
