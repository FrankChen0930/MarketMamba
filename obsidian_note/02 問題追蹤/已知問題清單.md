### SYS-12：WSL2 推論偶爾 60 分鐘超時、完全無輸出 ⚡ 部分緩解（2026-06-12）

- **問題**：6/2、6/10 兩次推論在 60 分鐘 `subprocess.run(timeout=3600)` 內完全沒有任何輸出（連 conda activate 訊息都沒有）就被砍掉，正常推論只需 9~14 分鐘
- **已加上**：
  - `_wsl_warmup()`：主推論前先跑 `wsl -d Ubuntu -e echo ok`（30 秒逾時），VM 沒反應就直接跳過本次推論
  - `_wsl_shutdown()` + 重試：第一次 60 分鐘逾時 → `wsl --shutdown` 重啟 VM → 等 10 秒 → 再跑一次（60 分鐘）
  - 所有時間點與耗時都印出明確時間戳
- **根本原因仍未確認**：6/10 當天使用者不在家、電腦無人操作（已排除遊戲/Blender 佔用 GPU）。懷疑方向：(a) 電腦睡眠後排程喚醒、WSL2 VM 冷啟動卡住 (b) Windows Update 背景安裝撞期 (c) Avira 排程掃描佔用磁碟 I/O (d) WSL2 VM 閒置過久喚醒卡死（已知通病）
- **下次卡住時**：比對 Windows 事件檢視器（Power-Troubleshooter / Kernel-Power 睡眠喚醒時間點）+ Windows Update / Avira 排程時間，並確認暖機/重試是否成功避開

### SYS-07：推論失敗無告警 ✅ 已解決（2026-06-12）

- **問題**：`run_daily_inference.py` 失敗時只寫 stdout log，沒有主動通知
- **解法**：`PersonalOS/scripts/run_daily.py` 新增 `send_telegram_notification()`，讀取 `.env` 的 `TELEGRAM_BOT_TOKEN`/`TELEGRAM_CHAT_ID`，於 `main()` 失敗區塊與 Windows Toast 並行發送，內容含各失敗任務狀態/耗時/錯誤摘要、是否觸發過 wsl 重試
- **分工**：Telegram = 後台自動化告警；PWA 推播（前端一般通知）使用者表示暫不需要，已擱置
- **待驗證**：下次真實失敗時確認通知格式可讀

### SYS-08：git push 失敗為 non-fatal，無 retry ✅ 已解決（2026-06-12）

- **問題**：`_push_to_github()` 失敗時繼續執行（non-fatal），原本**無自動重試機制**（`V6/run_daily_inference.py`，失敗直接 `return False`）
- **解法**：`git push origin main` 失敗時自動重試最多 3 次（間隔 10 秒），每次嘗試的成功/失敗都印出明確次數與錯誤訊息（截取前 200 字元）；3 次都失敗才回傳 `False` 並提示手動修復指令；`git add`/`git commit` 失敗仍視為非暫時性錯誤、不重試
- **待驗證**：觀察是否曾在 `inference.log` 中觸發過此重試
