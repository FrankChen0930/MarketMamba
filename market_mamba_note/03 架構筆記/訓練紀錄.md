- INPUT_DIM = 56

- scale_gate 沒有監控

## V6.2 第一次重訓（2026-06-02 開始）

### 改動內容

1. **GATv2 CSR 修正（PR 1）**：推論時改用 CSR 版本，GATv2 終於真正運作
2. **RS 相對強弱特徵**：RS_5d / RS_20d / RS_60d 加入 Group A
   - INPUT_DIM 56 → 59
   - 計算：Return_Nd - TWII_Return_Nd（TWII_Return_5d/20d/60d 在 _merge_macro 裡現算）
3. **scale_gate 監控**：每 5 epoch print Short/Mid/Long 比例

### 訓練過程

- Colab A100，每個 epoch 約 90 分鐘
- 有斷線，從 epoch 7（best）接續訓練
- 接續訓練的 scheduler 設計上刻意不還原（重建新的 OneCycleLR，RESUME_LR=5e-5，pct_start=0.05）

### 結果

- **best IC = 0.0950**（epoch 13），比 V6.1 的 0.0825 明顯提升
- RS 特徵效果比預期還明顯（在 train/val loss 幾乎貼合的情況下就達到 0.08+）
- epoch 13 之後出現 overfitting（val loss 上升，IC 下滑到 0.06 左右）→ 停止

### scale_gate 觀察

```
Short: 0.010  Mid: 0.001  Long: 0.989
```

> [!warning] 極端偏 Long
> Long branch 幾乎完全主導，Short 和 Mid 基本被廢棄。
> 原因：Long branch 使用完整 252 步，左側有 zero-padding，
> Mamba 把 padding 的零值當成信號學習，發現 Long branch 信號最穩定就全部押過去。
> 結果等於只用了一個純長期趨勢模型，RS 短期特徵根本沒被學到。

---

## V6.2 第二次重訓（啟用 Padding Mask）

### 改動內容

- `USE_PADDING_MASK = False` → `True`
- Long branch 的 Mamba 輸出在 padding 位置乘 0，截斷梯度
- Short / Mid branch 不需要 mask（它們用最後 20/60 步，全是真實資料）

### 目標

- 讓 scale_gate 從 Long 0.989 回到均衡（各 0.25~0.4）
- 讓 Short 和 Mid 的 RS 特徵真正被學到
- 預期 IC 有機會超過 0.10

### 觀察重點

**Epoch 5 的 scale_gate 數字是最重要的指標。**

## 下次重訓規劃方向

如果 scale_gate 均衡化之後 IC 還有空間：
- Alpha 因子剝離（剔除 Size、Momentum 系統性溢酬）
- 損失函數 VIX 加權（高波動環境下多學）
- Walk-Forward 驗證（V6.1 架構上的 WF，V5 的結論因特徵空間變化不能沿用）
