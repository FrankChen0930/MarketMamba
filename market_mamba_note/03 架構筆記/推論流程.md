## 目前已知的不穩定點

1. **WSL2 冷啟動競態**：Windows 重開機 → Task Scheduler 先觸發 → WSL2 還沒起來 → 直接失敗
2. **FinMind API 限速**：有重試保護，但連續失敗還是會炸
3. **git push 失敗沒有 retry**：失敗了就跳過，Render 會繼續用舊的快取
4. **沒有推播告警**：失敗只寫 stdout，不會主動通知
5. **VRAM OOM**：RTX 3060 12GB，2515 支股票推論有時候會爆

> [!warning] 失敗場景
> 任何一步失敗，Render 後端隔天還是提供昨日數據。
> Dashboard 有 freshness_warning（>3 天才顯示），1~2 天內的舊數據是看不出來的。

## ticker_universe.parquet 快取坑

這個快取很重要，有時候會污染整個 universe：

**症狀**：`[Dataset init] 46488 valid days | 46488 stocks pre-indexed`（正常約 2515 支）

**原因**：快取包含 `00400A`、`00679B` 等非 4 位數字代碼（ETF、債券等）

**修復**：
```python
Path("Data/cache_v6/ticker_universe.parquet").unlink()
# 順便過濾 prices_raw
df = pd.read_parquet("Data/processed_v6/prices_raw.parquet")
df = df[df["stock_id"].str.match(r"^\d{4}$")]
df.to_parquet("Data/processed_v6/prices_raw.parquet")
```
