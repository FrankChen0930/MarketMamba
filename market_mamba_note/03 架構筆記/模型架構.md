## 整體流程圖

```
輸入：(N_stocks, SEQ_LEN=252, INPUT_DIM=56)
  ↓
FactorGroupedEmbedding
  Group A price_momentum     (12 dims) → sub_dim 54
  Group B institutional_flow (20 dims) → sub_dim 94  ← 最大，機構流向最重要
  Group C fundamentals       (12 dims) → sub_dim 54
  Group D macro_environment  (12 dims) → sub_dim 54
  ↓
MultiScaleMambaEncoder（3 分支並行）
  Short Branch (20d)  — 2 層
  Mid Branch   (60d)  — 3 層
  Long Branch  (252d) — 3 層，有 Padding Mask（V6.2）
  → 自適應融合（scale_gate）
  ↓
GATv2（知識圖譜引導，CSR 稀疏矩陣，~640K 條邊）
  ↓
Gating Fusion
  gate = sigmoid(Linear(2×d_model → d_model))
  ↓
MultiHorizonHead
  → pred_5d / pred_20d / pred_60d
```

## 為什麼用 Mamba？

Mamba 的**選擇性遺忘機制**讓模型能動態決定「哪段歷史值得記住」。

對台股的意義：牛市 / 熊市 / 盤整的有效因子差異很大，固定注意力的 Transformer 很容易被不相干的歷史干擾。Mamba 理論上能更好地跟著市場狀態切換。

## 三個時間尺度的財金意義

| Branch | 天數 | 對應含義 |
|--------|------|----------|
| Short | 20d | 一個交易月，月線 KD 週期、月結後資金流動 |
| Mid | 60d | 季線，法說會週期、EPS 預期修正 |
| Long | 252d | 年線，完整財報週期、12-1 動能因子 |

這相當於同時看「5 分鐘、60 分鐘、日線」，但用機器學習自動融合。

## GATv2 知識圖譜

邊的種類和財金意義：

| 邊類型 | 權重 | 意義 |
|--------|------|------|
| 集團從屬 | 0.8 | 台積電生態圈、鴻海族群共漲共跌 |
| TPEX 供應鏈 | 0.6 | 上游行情往往領先下游 2~4 週 |
| TWSE 產業分類 | 0.5 | 同業輪動，但粒度不夠（子產業更細）|
| 60日滾動相關 | 動態 | 捕捉動態關聯，但對非線性關係有限制 |

**核心價值**：讓模型「知道」台積電法說會後，哪些供應鏈廠商應該同步被看漲。

## Padding Mask（V6.2）

- **問題**：Long branch 用完整 252 步，股票若不足 252 天，前面是 zero-padding
- **解法**：padding 位置乘 0，截斷梯度，確保 Long branch 只學真實資料的模式
- **Short / Mid 不需要**：分別取最後 20 / 60 步，在 ≥202 天的前提下全為真實資料

> [!note] 觀察假說
> Zero-padding 讓 Long branch 學到「開頭一串 0 → 後來有訊號」的假模式，導致它在 scale_gate 中過度主導。
> 加了 mask 後，Long 的梯度只來自真實資料，理論上 Short/Mid/Long 三個 gate 應該更均衡。

## 關鍵超參數

- `d_model = 256`
- `SEQ_LEN = 252`（一年交易日）
- `MC-Dropout = 30 次採樣`（估算不確定性）
- `Alpha 截斷 ±2.0`（防離群值）
- `Signal_Quality = Net_Alpha_20d / (Uncertainty + 1e-6)`，截斷至 [-10, +10]
