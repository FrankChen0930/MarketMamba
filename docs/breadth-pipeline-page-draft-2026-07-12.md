## 1. 資料層：三層 fallback 與覆蓋率

**修正計畫文件的一個錯誤**：`研究計畫_方向一` 寫的 fallback 是「Yahoo Finance → FinMind → 玉山證券」，實際（`fetcher.py` 檔頭與 `config.DATA_SOURCE_PRIORITY`）是：

| 層 | 來源 | 提供什麼 | 資料可用時間 |
|---|---|---|---|
| 1（最快） | yfinance | OHLCV 價量 | 收盤後 ~15 分鐘 |
| 2 | TWSE / TPEX 官方 API | 三大法人買賣超（T86 / dailyTrade） | ~16:30–17:00 |
| 3（fallback） | FinMind | 融資融券、千張大戶、月營收、財報等 | 18:00–19:00 |

玉山證券 API 只用於持倉同步（PersonalOS），不在研究資料管線內。

**Universe 與覆蓋**：
- 2005-01-03 至今、上市+上櫃 4 位數字代碼（`^\d{4}$` 過濾 ETF/權證/特別股）
- 門檻：市值 ≥ 5 億、5 日均成交值 ≥ 1,000 萬台幣（流動性下限＝實盤可執行性）
- 訓練 universe ~2,515 支 × ~5,200 個交易日；prices_raw 約 874 萬列
- 每日推論實際 cross-section ~1,950 支（低於 universe 的原因：新股 202 天歷史門檻 + 特徵 NaN 剔除，詳見 §7 限制）

**財金理由**：台股的資訊優勢很大一塊在籌碼面（三大法人、大戶持股），這些只有台灣本地源有；yfinance 只當價量的快速層。三層設計是為了「17:00 推論時每一類資料都已就位」，單一來源斷供時當日推論仍能跑。

## 2. Point-in-time 處理：對齊「市場何時知道」而非「數字屬於何時」

財報與月營收若按「所屬期間」對齊，模型會偷看未來（look-ahead bias）——3 月營收 4 月 10 日才公告，回測時 3 月 31 日就用它等於作弊。實作（`feature_engineer.py`）：

- **月營收**：`available_from = 當月月底 + 11 天`（台股規定 10 日前申報，留 1 天餘裕）
- **季報**：`available_from = 季底 + 45 天`
- 全部用 as-of join（`searchsorted` 向量化）：每個交易日只取「彼時已公告」的最新一筆
- 價量與法人買賣超為當日盤後資料，無此問題

**財金理由**：這是回測可信度的地板。任何 IC 數字，若 label 或特徵有 look-ahead，整個實驗作廢。本專案的 look-ahead 防護還包括：macro 時序標準化用 `shift(1)`（只用昨天以前的分布）、forward return 只存在於 label 欄且推論時不生成。

## 3. 特徵工程：59 維、4 因子組

原始因子 → 每日橫斷面 winsorize（1%/99%）→ 標準化 → 進模型：

| 組 | 維度 | 內容 | 為什麼是這組 |
|---|---|---|---|
| A 價格動能 | 15 | OHLCV、1/5/20 日報酬、MA、RSI、ATR、RS_5d/20d/60d（vs TWII 相對強度） | 動能與反轉是橫斷面選股最被驗證的因子族 |
| B 機構籌碼 | 20 | 外資/投信/自營買賣超、融資融券、借券、千張大戶持股與週變化、外資持股比 | 台股散戶佔比高，法人足跡的資訊含量在台股特別高——這是本系統對台股的主場優勢，維度給最多 |
| C 基本面 | 12 | PER、PBR、月營收 MoM/YoY、EPS、毛利率、ROE、FCF、殖利率等 | 估值與品質提供 20d/60d horizon 的慢變量 |
| D 總經環境 | 12 | TWII/SPX 報酬、VIX、美債殖利率、金油匯、期貨未平倉、P/C ratio、景氣燈號、FED 利率 | 控制 regime：同一根 K 線在多頭與空頭的意義不同 |

**標準化的兩套規則**（教訓換來的設計，見 §7 D1）：
- 股票級特徵（A/B/C）：per-date 橫斷面 z-score ± 3σ 截斷——模型要學的是「今天誰相對強」，不是絕對值
- 總經特徵（D）：expanding time-series z-score（shift(1)、最少 252 天）——macro 同日全市場同值，橫斷面 z-score 會把它整組歸零

**Label**：`Alpha_Nd = 個股前瞻 N 日累積報酬 − TWII 同期報酬`，訓練時再轉每日橫斷面百分位 rank（理由見 §5）。

## 4. 模型輸入規格

- 輸入張量：`(N_stocks ≈ 1,950, SEQ_LEN = 252, INPUT_DIM = 59)`——每支股票一整年的日頻特徵序列
- 納入門檻：至少 202 天（252 × 0.8）真實歷史；不足 252 天者前段 zero-padding + mask
- 知識圖譜（GATv2 用）：**642,451 條邊**（`graph_builder.py` 建構、CSR 快取），四種邊型態與權重：集團從屬 0.8、產業鏈 0.6、TWSE 產業分類 0.5、60 日滾動相關 ≥ 0.7（動態）
- Lazy loading：tensor 在 `__getitem__` 才建，一個 batch = 一整個交易日的 cross-section

**財金理由**：252 天覆蓋一個完整年度循環（財報四季 + 除權息），KG 讓「台積電漲、供應鏈跟漲」這種結構性傳導不必靠模型自己從價格猜。

## 5. 模型架構：為什麼是 Mamba + GATv2 + rank 目標

```
FactorGroupedEmbedding（4 組按維度比例投影 → d_model 256）
  → MultiScaleMambaEncoder（短 20 步 / 中 60 步 / 長 252 步三分支，softmax gate 自適應融合）
  → GATv2（1 層 4 heads，KG 邊引導的橫斷面訊息傳遞，每股鄰居上限 15）
  → Gating Fusion（sigmoid gate 混合時序/圖表徵）
  → 3 個獨立線性頭 → Alpha_5d / 20d / 60d
約 4M 參數；MC-Dropout 30 次採樣輸出每股 Uncertainty
```

**為什麼 Mamba 而非 Transformer/LSTM**：252 步長序列下，自注意力是 O(L²)，SSM 是 O(L)；在「每天要對 ~2,000 支股票各跑一次 252 步序列」的推論負載下（本機 RTX 3060），線性複雜度是可行性差異而非品味差異。與 LSTM 的實證差距由方向二 baseline 對照回答（誠實聲明：在對照結果出來前，「Mamba 優於簡單序列模型」是假設不是結論）。

**為什麼 GATv2**：橫斷面關係（產業、集團、供應鏈）是台股 alpha 的結構性來源，靠時序模型從價格資料重新發現太浪費。實證：GAT 修復前後（§7 M1），最終排名 Top50 重疊只有 30/50、Uncertainty 整體 −34%——圖的貢獻是可度量的，不是裝飾。

**為什麼 rank 目標 + ranking loss（本頁最重要的論述）**：
評估指標是 Spearman IC（排名相關），但舊版訓練目標是 z-score 報酬的 MSE——優化目標與評估目標錯位。台股漲跌幅 ±10% 限制下報酬分布厚尾，MSE 會被極端漲跌股主導：模型花容量去精確預測暴漲股「漲多少」，而我們只需要它排對「誰比誰強」。消融證據（依時間序）：
- Phase 0（5d、無 ranking loss、z-score 目標）：5d IC **0.0434**
- Phase 1（+ listnet_5d=0.5，其餘不變）：5d IC **0.0487**（+12%，代價是過擬合加劇）
- Phase 2（目標改每日橫斷面 rank + listnet）：5d IC **0.0951**（對 0.0434 **近乎翻倍**，乾淨 out-of-sample：train ≤2023、val 2024–26）
即：把「優化什麼」對齊「評估什麼」的貢獻，比任何架構調整都大。這也是為什麼 §3 的 label 要做 rank 轉換。

**多尺度的誠實註記**：deep supervision 診斷顯示三個時間分支對 5d 預測冗餘（三分支輔助 IC 全擠在 ~0.05、融合無增益），短線模型已因此簡化為單尺度（60 步、1.66M 參數）；趨勢模型的 gate 在 ep3 即收斂至 Long 分支 100%。「多尺度自適應融合」的實際結論是**它教會我們何時不需要多尺度**——這是誠實的研究產出，詳見 §7。

## 5.5 Baseline 對照：用可解釋性換到多少效益？（方向二 Step 5 交付，2026-07-15 嵌入）

同一協定（同 universe / rank label / 切分 / 成本）下的四階對照，test = 2024-01 ~ 2026-06 共 580 交易日。完整定稿與引用紀律見 `docs/baseline-comparison-table-2026-07-15.md`：

| 5d | Ridge | GBDT | GRU | Mamba v6_short |
|---|---|---|---|---|
| 輸入 | 300 維扁平 | 300 維扁平 | (60,59) 序列 | (252,59) 序列 + KG |
| 參數量 | 301 | ~10⁴ 葉 | ~49K | 1.66M |
| IC 全市場 / 高流動 | +0.1015 / +0.0705 | +0.1098 / +0.0802 | **+0.1113 / +0.0867** | +0.0870 / — |
| 組合層年化（Top50 等權） | +18.7% | +10.8% | **+22.8%** | 未跑 |
| 成本×2 年化 | −5.4% | −13.6% | −2.6% | — |
| 可解釋性 | **係數可讀** | SHAP | 無 | 無 |

**答案：在 5d 這條線上，可解釋性的代價是負的**——最可解釋的 Ridge 已勝 Mamba，往黑箱走的增量合計僅 +0.010。三條獨立證據收斂：①模型形式只在 ±0.01 內移動，特徵工程（+0.015）與 rank label（近翻倍）才是大頭；②序列 vs 扁平幾乎無差（GRU vs GBDT +0.0015）；③ ~49K 參數 GRU 同特徵同 window 勝 1.66M 參數 Mamba +0.024，架構紅利不成立。**同時保留 GBDT 教訓：IC 排名 ≠ 落袋排名**（GBDT IC 較高、組合層反輸 Ridge 7.9pp），所以訊號層與組合層必須並排看。引用紀律：headline 需配分層數字（全市場被小型股墊高）、存活者偏差未量化、成本×2 全階轉負屬高換手策略固有屬性。

這不推翻 DL 線的其他價值（60d、Uncertainty 校準、KG 事件傳導未對照），但證明本專案的研究流程能誠實回答「複雜度有沒有換到東西」。

## 7. 誠實限制與弱點（加分段，全部有據可查）

1. **GAT 曾在推論時失效近 6 週（M1，已修復）**：為省 VRAM 分批推論，每批只取批內 KG 邊，跨批邊全丟——GAT 推論時近乎退化成 identity，與訓練（全圖）嚴重不一致。2026-06-12 改兩段式推論修復；修復當日 Top50 重疊僅 30/50、Uncertainty −34%，證明先前排名確實被此 bug 扭曲。
2. **Macro 特徵組曾整組歸零（D1，部分修復）**：總經特徵同日全市場同值，橫斷面 z-score 除以 std=0 → 整組 12 維恆為 0，模型 macro 分支長期無資訊輸入。雙模型已改 time-series 標準化；**V6.1（線上 20d 主力）尚未套用**，其 macro 分支至今實質無效。
3. **存活者偏差（D3，未修復）**：universe 取自當前上市清單，已下市股不在訓練資料——模型沒看過「走向下市的股票長什麼樣」，對地雷股的辨識力存疑。完整修復需歷史成分股名單，現階段靠流動性門檻降低實害並在此誠實揭露。
4. **每日有效 cross-section ~1,950/2,515**：202 天歷史門檻 + NaN 剔除。新上市一年內的股票模型不覆蓋。
5. **Scale gate 塌縮**：趨勢模型 gate ep3 即 100% 押 Long 分支；5d 目標下 Short 分支近 0（診斷顯示是「被 gate 餓死」而非無資訊）。多尺度設計對單一 horizon 的增益不成立，短線線已改單尺度。
6. **Uncertainty 校準是進行中的研究，不是已解決的功能**：MC-Dropout U 與後續 |超額報酬| 相關約 +0.20~0.25（抓得到波動幅度）；「低 U = 更準」在 20d 大致成立（U 五分位箱內 IC：最有信心箱 0.120、最沒把握兩箱 0.083–0.089，遞減趨勢但非嚴格單調）、**在 5d post-P0 樣本上暫時反向**（僅 14 天，7 月底樣本夠再判，詳見 `docs/conviction-c-analysis-2026-07-11.md`）。
7. **總經 raw 資料自 2026-04-24 未更新**：每日更新不含 macro，保守模式 regime 閘門實質未啟用（對 V6.1 推論無影響，原因見第 2 點的諷刺現實）。
8. 線上 IC 樣本仍淺（20d 僅個位數天數）、WF 未例行化（見 §6）——頁面所有績效數字都附樣本天數，樣本不足處明講。
